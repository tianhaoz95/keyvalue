# Tasks

## 1. Persistent Local Mode ("No Sign In")
Implement a fully functional, offline-first experience that doesn't require Firebase Auth.

- [ ] **Infrastructure: Add Local Database**
    - [ ] Add `hive` and `hive_flutter` to `pubspec.yaml` for lightweight, reactive local storage.
    - [ ] Initialize Hive in `main.dart`.
    - [ ] Create Hive Adapters for `Cpa`, `Customer`, and `Engagement` models.

- [ ] **Data Layer: Local Repositories**
    - [ ] Implement `LocalCpaRepository`, `LocalCustomerRepository`, and `LocalEngagementRepository` using Hive.
    - [ ] Ensure they follow the same interface/pattern as existing Firestore repositories.
    - [ ] **Migration Logic**: Update `CpaProvider` to toggle between `FirebaseRepos` and `LocalRepos` based on the user's login state (e.g., `local_user` UID).

- [ ] **UI: "Continue as Guest" Workflow**
    - [ ] Update `LoginScreen` to replace "Demo Mode" with a "Continue as Guest" (Local Mode) option.
    - [ ] In Local Mode, the app should save all data to the local database instead of returning static mock data.

## 2. Platform Branding & Asset Generation
Standardize the app's visual identity across all platforms using the raw design asset.

- [ ] **Asset Generation (Web & Desktop)**
    - [ ] Use `magick` (or `convert`) to generate `app/web/favicon.png` (32x32).
    - [ ] Generate a high-res `app/assets/images/logo_icon.png` (1024x1024) for desktop icons.

- [ ] **Mobile App Icons (Android & iOS)**
    - [ ] Generate the necessary icon set for Android (`mipmap` densities).
        - [ ] Square icons (48x48, 72x72, 96x96, 144x144, 192x192).
        - [ ] Rounded icons.
    - [ ] Generate the iOS AppIcon set (multiple sizes from 20x20 to 1024x1024).
    - [ ] **Tooling**: Consider adding `flutter_launcher_icons` to `pubspec.yaml` to automate this process from a single high-res source generated by `magick`.

- [ ] **Theme Integration**
    - [ ] Ensure the generated icons use the `primaryIndigo` (#1A237E) background for a consistent "Premium Professional" look.
